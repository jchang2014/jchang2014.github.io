About Blocks, Procs, and Lambdas in Ruby.

April 25, 2015

So far, we've been using a lot of iterating methods in our challenges. They all have the similar syntax of using either curly brackets or the "do...end" enclosure. The code that lies in between is called the block. For example, these highlighted portions are the block:

meal = ["Whopper", "Fries", "Coke"]
meal.each {|item| print "I want an extra large #{item}! "}
#=>I want an extra large Whopper! I want an extra large Fries! I want an extra large Coke!

Now, Procs, are convenient tools we can use to save blocks. In this way, we can kind of think of procs as variables in which we store the blocks that we want to run. For example, if we had multiple meals that we wanted to super size, we could do this:

meal1 = ["Whopper", "Fries", "Coke"]
meal2 = ["Big Mac", "Chicken Nuggets", "Sweet Tea"]

super_size_me = Proc.new {|item| print "I want an extra large #{item}! "}

meal1.super_size_me
#=>I want an extra large Whopper! I want an extra large Fries! I want an extra large Coke!

meal2.super_size_me
#=>I want an extra large Big Mac! I want an extra large Chicken Nuggets! I want an extra large Sweet Tea!

Finally, there are Lambdas, which are essentially the same as Procs, but are less controlling in methods. When you use Procs, they take over the method you call them in, but when you use Lambdas, they take care of their business and let the method return to whatever it was doing. Let's look at an example:

def proc_method
  Proc.new { return "I'm a Proc!"}.call
  return "Now back to you!"
end

def lambda_method
  lambda {return "I'm a Lambda!"}.call
  return "I was never here!"
end

puts proc_method #=> I'm a Proc!
puts lambda_method #=> I was never here!

As you can see, the Proc completely took over the method and ended as soon as it was done with what it had to say. The lambda on the other hand, finished what it had to say, and allowed the method to continue. Hopefully, with these two useful tools in our belt, we can continue on our trip through Ruby as more capable and intelligent developers!
