I'm feeling a little Loopy...

About loops in Ruby vs. loops in JavaScript.

May 3rd, 2015

As we all know by now, loops are arguably one of the most useful tools we have at our disposal for creating functions. Both Ruby and JavaScript allow us to create loops that allow us to perform a code block multiple times efficiently. For example, both have a method called "while" that continues the loop as long as a condition is still true. Here we have Ruby on the left, and JavaScript on the right. As you can see, both have pretty similar logic, but the syntax is slightly different.

Now let's move on to "for" loops. This is where we start to see a difference between Ruby and JavaScript. In JavaScript, our "for" loops are really limited to the same sort of counter logic, where we have a variable set to a certain integer, and we either add or remove from that integer to continue the loop. However, JavaScript gives us a convenient single-line syntax to do this with. In Ruby, we can do a similar logic, but using a range of numbers from x..y. As you can see, both languages again have similar logic, with slightly different syntax. However, Ruby allows for a bit more flexibility in "for" loops. We can tell it to loop through each item in the array simply by defining what we're looking for.

In addition to this flexibility, Ruby also gives us an extra method called "until" that works similarly to while, but instead of operating the loop so long as the condition is true, it will continue the loop so long as the condition is false. Hopefully now you can see the similarities and differences between Ruby loops and JavaScript loops. Try them out!
